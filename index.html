Introduction to CSS Position Property

Introduction Static Position Fixed Position Sticky Position Absolute vs. Relative Position

Introduction

The position property specifies the type of positioning used for an element, in total there are five different position values, they are:

  • Static
  • Relative
  • Fixed
  • Absolute
  • Sticky

Using these different position properties can be adjusted using the top, bottom, left and right properties. It is mandatory that you set the value as so.

In example, you would set the element position property using css. then set left, right, top, bottom. See example below:

element {
position: fixed;
top: 10px
}

We'll talk more about this in detail. In later sections.

Static Position

By default, elements are static by default. That being said, elements with the static position are not able to be positioned with left, right, top and bottom.

element{
position: relative //remember by default, the position is relative, unneeded unless resetting position to default.
}

Fixed Position

A fixed position is always in respect to the viewport for the user. So for example

For example, when the elements has the top of 0, then the position will always be at the top of the page regardless of the position the viewer has scrolled to. Think about this in contrast to the sticky position property explained below.

element {
position: fixed;
top: 0px; //Affixes element to the top of the page.
}

Sticky Position

The sticky position is based on the user's scroll position. A sticky elements toggles between relative and fixed, dependent on the scroll position by the user's viewport.

For example, when the elements hits the limit of the viewport. The element will become fixed at that given position with the sticky position. The element will follow the user's position at a fixed position becoming permanent.

position: sticky; //Once limit of viewport has been reached upon element, sticky will attach to viewport at top of page. top: 0px;

Absolute vs. Relative Position

Although distinct position properties, Absolute and Relative Position function in similar manner. To best explain both of the properties, it'd probably be most beneficial to contrast and compare the two.

Relative position moves the box model of the element the space "relative" to the element's current position.

For example, let's call the box element 1 and it has the class element 1.
.element 1 {
position: relative;
top: 20px; // Current position top would be increased by the 20px with a gap left for the user's old position.
}

Absolute is the opposite, it does not take into account where the element will be and would instead just base it's position on the parent's container.
